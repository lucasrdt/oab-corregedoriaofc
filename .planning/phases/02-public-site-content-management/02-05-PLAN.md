  <name>Task 0: Diagnosticar Supabase Storage no Dashboard</name>
  <what-built>Diagnóstico manual necessário — Claude não tem acesso ao Supabase Dashboard</what-built>
  <how-to-verify>
    1. Acesse o Supabase Dashboard do projeto OAB-MA
    2. Vá em "Storage" no menu lateral esquerdo
    3. Verifique se o bucket `site-assets` existe:
       - SE NÃO existe: clique "New bucket", nome: `site-assets`, marque "Public bucket" → Create
       - SE existe: verifique se está marcado como "Public" na lista de buckets
    4. Se o bucket estava privado, mude para público: clique no bucket → "Edit bucket" → marque "Public"
    5. Vá em "Policies" do bucket `site-assets` e verifique se existem policies:
       - Policy de leitura pública (SELECT para todos): se não existe, adicione executando no SQL Editor:
         ```sql
         CREATE POLICY "Public read site-assets" ON storage.objects
           FOR SELECT USING (bucket_id = 'site-assets');
         ```
       - Policy de upload autenticado (INSERT): se não existe:
         ```sql
         CREATE POLICY "Auth upload site-assets" ON storage.objects
           FOR INSERT WITH CHECK (bucket_id = 'site-assets' AND auth.role() = 'authenticated');
         ```
       - Policy de delete autenticado (DELETE): se não existe:
         ```sql
         CREATE POLICY "Auth delete site-assets" ON storage.objects
           FOR DELETE USING (bucket_id = 'site-assets' AND auth.role() = 'authenticated');
         ```
    6. Verifique o arquivo `.env.local` na raiz do projeto — confirme que `VITE_SITE_ID` está definido com o UUID correto do site OAB-MA (encontrado na tabela `sites` do Supabase)

    Anote qual era o problema (bucket não existia / estava privado / policies faltando / VITE_SITE_ID ausente) para incluir no SUMMARY.
